Introduction

ZachSite is my first website. I intend it as a place for me to showcase personal code projects, especially web applications. It's kind of a barebones layout, but I wanted to get experience with database driven web development and certain frameworks/toolsets that I plan to use in the future, so under the hood it is maybe a little more robust than it needs to be. This page is just me bragging about all those goodies with which I now have some modest experience. So, if that interests you, read on.

The Server Side

Built With:

  • Django, the Python web framework

  • PostgreSQL database

Django is really the key framework of the project, and learning how to use it effectively and properly has been my main focus through the whole development process. Also, I think databases are neat, so I have tried to make the site as database driven as possible. For example, the documentation pages for my non-web project(s) (see AudioIO) are composed almost entirely dynamically from the database so that I can easily edit a class description or method signature without having to go into the HTML. Unit testing is performed with Django's extension of the Python standard library unittest module.

The Client Side

Built With:

  • Twitter Bootstrap, the mobile-first front end framework

  • Sass, the CSS extension language

  • Grunt, the Node.js task runner. Used mainly for compiling SCSS files and running QUnit unit tests

UX design is not my primary interest. Take it at face value. However, I have enjoyed getting my hands dirty with some of the CSS and Javascript. I've used Bootstrap as the real workhorse, but I've tried to customize the look and feel in ways that I think create value. I don't know, you tell me.... UI level testing so far has been manual, with thorough testing on latest OSX and iOS Safari, and smoke testing on OSX FireFox.
